### 1.1. Архитектура ОС GNU/Linux. Дистрибутивы.

Linux — это **монолитное ядро с элементами микроядерной архитектуры** (модульность).

**Уровневая структура:**

1. **Аппаратное обеспечение** (Hardware)
   - Процессор, память, диски, периферия

2. **Ядро (Kernel)** — Сердце системы, разработка Линуса Торвальдса
   - **Драйверы устройств** (как встроенные, так и загружаемые модули)
   - **Управление памятью и процессами**
   - **Системные вызовы (syscalls)** — интерфейс между ядром и пользовательским пространством
   - **Сетевой стек**
   - **Файловые системы** (ext4, Btrfs, XFS и др.)

3. **Пользовательское пространство (User Space)** — в основном проекты GNU
   - **Библиотеки**: glibc (стандартная библиотека C)
   - **Системные утилиты**: coreutils (ls, cp, rm), bash, gawk и сотни других
   - **Демоны (сервисы)**: systemd/init, cron, сетевые службы

4. **Пользовательские приложения**
   - Графические оболочки (GNOME, KDE)
   - Офисные пакеты, браузеры, медиаплееры

**Ключевая философия**: "Всё есть файл" (дескрипторы, устройства, сокеты представлены как файлы).

## 2. Дистрибутивы (Distributions)

Дистрибутив — это **комплектация** ядра Linux, программ GNU, дополнительного ПО и системы управления пакетами.

**Ключевые компоненты дистрибутива:**
- Ядро Linux (часто с патчами)
- Пакеты GNU и другие открытые программы
- Менеджер пакетов (dpkg/RPM/Pacman и др.)
- Система инициализации (systemd, OpenRC, runit)
- Инсталлятор
- Документация

## 3. Исторические основы дистрибутивов ("генеалогическое древо")

### **Корневые (исторически значимые) дистрибутивы:**

1. **Slackware** (1993) — старейший живой дистрибутив
   - Простота, близость к UNIX-философии
   - **Потомки**: SUSE (через SLS)

2. **Debian** (1993) — философия свободы, сообщество
   - Система пакетов: **.deb** и менеджер **dpkg/APT**
   - **Основные ветки-потомки**:
     - **Ubuntu** (2004) → самый популярный десктоп-дистрибутив
       - → **Linux Mint** (основан на Ubuntu/Debian)
       - → **Elementary OS**, **Pop!_OS**, **Zorin OS**
     - **Knoppix** → первый LiveCD
     - **Raspbian/Raspberry Pi OS** (для Raspberry Pi)

3. **Red Hat Linux** (1994) → коммерческий фокус
   - Система пакетов: **.rpm**
   - **Основные ветки**:
     - **Red Hat Enterprise Linux (RHEL)** (2003) → коммерческий, стабильный
       - → **CentOS** (клонированная RHEL, теперь под контролем Red Hat)
       - → **Scientific Linux** (устарел)
       - → **Oracle Linux**
       - → **Rocky Linux** (преемник CentOS после изменений 2020)
       - → **AlmaLinux** (аналогично Rocky)
     - **Fedora** (2003) → инновационный upstream для RHEL

4. **Arch Linux** (2002) — принцип KISS, rolling release
   - Менеджер пакетов: **Pacman**, AUR
   - **Потомки**: Manjaro, EndeavourOS, Garuda Linux

5. **Gentoo** (2002) — компиляция из исходников
   - Менеджер пакетов: **Portage**
   - **Потомки**: Chrome OS (ядро), Funtoo

### **Независимые/особые дистрибутивы:**
- **openSUSE** (изначально из Slackware, затем свой путь)
- **Mandriva** (устарел) → **Mageia**, **OpenMandriva**
- **Android** (ядро Linux сильно модифицировано)
- **NixOS** — декларативная конфигурация, воспроизводимость

## 4. Основные семейства сегодня (2020-е)

1. **Debian-семейство** (Ubuntu и производные) — доминирует на десктопах и серверах
2. **RHEL-семейство** (Fedora, CentOS/Rocky/Alma) — корпоративные серверы
3. **Arch-семейство** — для энтузиастов, rolling release
4. **openSUSE** — свой экосистема (YaST, OBS)
5. **Независимые** (Slackware, Gentoo, NixOS) — для специфических задач

## Эволюционные тренды:
- **Контейнеризация**: Docker/Kubernetes способствовали конвергенции
- **Systemd**: стал стандартом инициализации (кроме некоторых дистрибутивов)
- **Flatpak/Snap**: кросс-дистрибутивные пакеты
- **Immutable-дистрибутивы**: Fedora Silverblue, openSUSE MicroOS

Эта архитектура и экосистема дистрибутивов создала уникальную ситуацию: **единое ядро + разнообразие пользовательских сред**, что обеспечивает и стабильность, и гибкость.

## 1.2. Особенности, дистрибутивы и уровни защищенности Astra Linux Special Edition (SE)

### **1.1. Сертификация и назначение**
- **Высший уровень защиты** в России: сертифицирована **ФСТЭК, ФСБ, Минобороны**
- **Максимальный класс защиты**: 1У (первые части) и 1Б (вторые части) по руководящим документам Гостехкомиссии
- Предназначена для **госорганов, силовых структур, оборонных предприятий, банков**
- Поддерживает работу с **совершенно секретными сведениями**

### **1.2. Архитектурные особенности защиты**
- **Мандатное управление доступом (Мандатный контроль доступа - МКД)**
  - Каждому субъекту (пользователю) и объекту (файлу) присваивается **уровень конфиденциальности** (метка)
  - Поток информации строго контролируется: "снизу вверх" запрещен
- **Дискреционное управление доступом** (стандартное UNIX-права)
- **Ролевая модель доступа (РМД)**
  - Администратор разделен на роли:
    - **Администратор безопасности** (настройка политик)
    - **Администратор аудита** (просмотр логов)
    - **Системный администратор** (обслуживание)
    - **Администратор пользователей**
- **Изолированный программный контур (ИПК)**
  - Возможность работы с информацией разного уровня в одной системе
  - Разделение по сменным носителям, виртуальным машинам

### **1.3. Технические особенности**
- **Пакет средств защиты (ПСЗ) "Стрелец"** — комплекс ПО для обеспечения требований защиты
- **Модифицированное ядро Linux** с включенными патчами безопасности (SELinux/AppArmor аналоги + собственные разработки)
- **Полное аудирование** всех действий (невозможно отключить)
- **Криптографическая защита** (встроенные средства, совместимость с российской криптографией)
- **Запрет загрузки с внешних носителей** по умолчанию

## 2. Дистрибутивы (редакции) Astra Linux

### **2.1. Основные линейки:**
1. **Astra Linux Common Edition (CE)** 
   - Для общего применения (без требований к защите гостайны)
   - Сертифицирована ФСТЭК для защиты персональных данных

2. **Astra Linux Special Edition (SE)** — **главный продукт**
   - Полный набор средств защиты
   - Редакции внутри SE:
     - **"Смоленск"** (x86-64) — основная
     - **"Орел"** (ARM, российские процессоры: Эльбрус, Байкал, Фишкойн)
     - **"Воронеж"** (PowerPC, для импортозамещения IBM-серверов)

3. **Astra Linux Embedded Edition**
   - Для встраиваемых систем (киоски, терминалы)

### **2.2. Графические окружения:**
- **Fly** (собственная разработка) — основной интерфейс в SE
- **GNOME** — в CE и как опция в SE
- **Unity** (старые версии) — теперь сменен на Fly

## 3. Уровни защищенности (классы)

### **3.1. По приказу ФСТЭК №17 (системы виртуализации):**
- **Уровень 1** — максимальный (для особо важных объектов)
- **Уровень 2** — повышенный
- **Уровень 3** — базовый
- **Уровень 4** — минимальный

**Astra Linux SE** соответствует **уровням 1-3** в зависимости от конфигурации.

### **3.2. По руководящим документам (РД) Гостехкомиссии:**
- **Класс 1У** — системы, обрабатывающие информацию **со степенью секретности "совершенно секретно"**
- **Класс 1Б** — системы для **"секретно"**

**Astra Linux SE сертифицирована по обоим классам.**

### **3.3. Уровни доверия (по ГОСТ Р ИСО/МЭК 15408):**
- **Уровень доверия к безопасности (УДБ) 4** — самый высокий для коммерческих ОС
- **Evaluation Assurance Level (EAL) 4+** в международной классификации

## 4. Практические аспекты использования

### **4.1. Отличия от обычных Linux-дистрибутивов:**
- **Сложность настройки** — требует обученного администратора безопасности
- **Пониженная производительность** из-за постоянного аудита и проверок
- **Ограниченный софт** — только ПО, прошедшее проверку и сертификацию
- **Жесткая политика обновлений** — только утвержденные обновления

### **4.2. Типовые сценарии применения:**
- **Автоматизированные рабочие места (АРМ)** в госорганах
- **Серверы обработки конфиденциальной информации**
- **Системы управления технологическими процессами (АСУ ТП)** на критически важных объектах
- **Корпоративные сети банков и финансовых организаций**

### **4.3. Репозитории и обновления:**
- **Основной репозиторий** — только проверенные пакеты
- **Режим "заморозки"** — после сертификации состав ПО не меняется
- **Центр обновления "Астра-Центр"** — управление обновлениями

## 5. Сертификаты и нормативная база
- **Сертификат ФСТЭК России** № 4068 (на СВТ)
- **Сертификат Минобороны России**
- **Включена в Единый реестр российских программ** (№ 1981)
- **Соответствует требованиям:**
  - Приказ ФСТЭК №17, 21, 31
  - Приказ ФСБ №514, 795
  - ГОСТ Р 50739-95, ГОСТ Р ИСО/МЭК 15408

## 6. Критика и проблемы
- **Высокая стоимость** лицензий
- **Сложность администрирования**
- **Несовместимость** с некоторым современным ПО
- **Производительность** ниже, чем у обычных дистрибутивов
- **Ограниченная поддержка оборудования**

**Итог:** Astra Linux SE — это **не просто дистрибутив Linux, а комплексная система защиты**, где ОС является лишь одним из компонентов. Её использование оправдано только там, где есть реальные требования по защите государственной тайны или коммерческой информации особой важности. Для обычных задач (офис, разработка) существенно удобнее использовать Common Edition или другие дистрибутивы.
